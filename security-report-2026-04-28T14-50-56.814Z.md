recommended_actions:
1. Заблокируйте/ограничьте трафик с подозрительных IP (особенно участвующих в /wp-admin/install.php и path traversal payload) на уровне WAF/ACL; включите rate limiting.
2. Проверьте сервер/приложение на уязвимости: WordPress hardening (отключение install endpoint), защита от directory traversal/RCE в CGI, корректность обработки /cgi-bin/ и нормализация URL.
3. Убедитесь, что .env и другие секреты не доступны через web root; проверьте Nginx root/alias, запреты на доступ к файловым путям, добавьте deny для чувствительных расширений/точек.
4. Проверьте наличие попыток эксплуатации Exchange и примените соответствующие патчи/mitigations (если сайт связан с Exchange/внешними интеграциями).
5. Настройте детект бинарного мусора/неправильного протокола: отдельные правила для бинарных строк в request и усиление логирования/блокировки при повторяемости 400/157.
6. Снимите метрики/индикаторы компрометации: логи доступа/ошибок за период кампании, изменения в конфигурациях, наличие новых файлов/cron, состояние upstream сервисов.
7. Проведите ретроспективный анализ API авторизации/ошибок на domen.ru (POST /api/v1/auth/telegram/get-code и ответы 400/401) и проверьте наличие brute-force/enum-поведения.
